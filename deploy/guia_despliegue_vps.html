Prerequisitos antes de empezar

  • Acceso SSH al VPS como root (o usuario con sudo)
  • DNS: siceback.maewalliscorp.org → IP del VPS (propagado)
  • DNS: sice.maewalliscorp.org → IP del VPS (propagado)

ℹ️
Verifica que los DNS estén propagados antes del paso 7 (SSL). Puedes comprobarlo con: dig siceback.maewalliscorp.org +short — debe devolver la IP del VPS.
1
Preparar archivos en tu Mac

1.1 Compilar el frontend

Ejecuta esto en tu terminal local antes de subir los archivos:

# Ir al directorio del frontend
cd /Users/edegantea/development/itsmt-sice/frontend

# Crear variables de entorno de producción
echo "VITE_API_URL=https://siceback.maewalliscorp.org/api" > .env.production

# Compilar (el resultado queda en frontend/dist/)
npm ci && npm run build
✅
Si el build termina sin errores, verás frontend/dist/ con index.html y carpeta assets/. Esa carpeta completa es lo que va al VPS.

1.2 Qué subir y qué excluir

Qué subirDestino en el VPS
Carpeta backend/ completa/var/www/sice-backend/
Contenido de frontend/dist//var/www/sice-frontend/
Excluir al subirRazón
backend/vendor/Se instala en el servidor con Composer
backend/.envSe crea manualmente en el servidor
backend/storage/logs/*.logLogs locales de desarrollo
backend/bootstrap/cache/Se regenera en el servidor
node_modules/No se usa en producción

1.3 Subir con rsync (recomendado)

# Backend
rsync -avz --progress \
  --exclude='.env' \
  --exclude='vendor/' \
  --exclude='storage/logs/*.log' \
  --exclude='bootstrap/cache/' \
  /Users/edegantea/development/itsmt-sice/backend/ \
  root@<IP_VPS>:/var/www/sice-backend/

# Frontend (el contenido de dist/, no la carpeta dist en sí)
rsync -avz --progress --delete \
  /Users/edegantea/development/itsmt-sice/frontend/dist/ \
  root@<IP_VPS>:/var/www/sice-frontend/

Configuración inicial del servidor

Conéctate al VPS: ssh root@<IP_DEL_VPS>

2.1 Actualizar el sistema

apt update && apt upgrade -y
apt install -y curl wget git unzip software-properties-common ufw fail2ban

2.2 Instalar PHP 8.3

add-apt-repository ppa:ondrej/php -y
apt update
apt install -y \
  php8.3 php8.3-fpm \
  php8.3-pgsql php8.3-mbstring php8.3-xml php8.3-bcmath \
  php8.3-curl php8.3-zip php8.3-gd php8.3-intl \
  php8.3-tokenizer php8.3-ctype php8.3-fileinfo php8.3-dom

# Verificar
php8.3 -v

2.3 Instalar PostgreSQL 16

sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
apt update && apt install -y postgresql-16 postgresql-client-16
systemctl enable postgresql && systemctl start postgresql

2.4 Instalar Nginx

apt install -y nginx
systemctl enable nginx && systemctl start nginx

2.5 Instalar Composer

curl -sS https://getcomposer.org/installer | php -- --install-dir=/usr/local/bin --filename=composer
composer --version

2.6 Instalar Docker

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
  > /etc/apt/sources.list.d/docker.list
apt update && apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
systemctl enable docker && systemctl start docker

2.7 Instalar Certbot

apt install -y certbot python3-certbot-nginx

2.8 Configurar Firewall (UFW)

ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw --force enable
ufw status
⚠️
Asegúrate de que el puerto 22 (SSH) esté permitido ANTES de activar UFW. El comando ufw allow OpenSSH lo hace automáticamente.

Base de datos PostgreSQL

3.1 Crear usuario y base de datos

sudo -u postgres psql

Dentro de psql, ejecuta (reemplaza TU_CONTRASEÑA_SEGURA por una contraseña real):

CREATE ROLE sice_user LOGIN PASSWORD 'TU_CONTRASEÑA_SEGURA';
CREATE DATABASE sice_produccion
  OWNER sice_user
  ENCODING 'UTF8'
  LC_COLLATE 'C'
  LC_CTYPE 'C'
  TEMPLATE template0;
GRANT ALL PRIVILEGES ON DATABASE sice_produccion TO sice_user;
\q

cd /var/www/sice-backend

# Generar APP_KEY (esto edita el .env automáticamente)
php artisan key:generate

# Dar permisos a www-data para escribir en storage
chown -R www-data:www-data storage bootstrap/cache
chmod -R 775 storage bootstrap/cache

# Crear enlace simbólico para uploads (logos, imágenes)

⚠️
Si ves chown: Operation not permitted en algún archivo, es porque el usuario actual no es propietario. Ejecuta primero: chown -R $USER:$USER /var/www/sice-backend y luego vuelve a ejecutar el chown a www-data.

6
Configurar Nginx

6.1 Configuración del backend

nano /etc/nginx/sites-available/sice-backend
server {
    listen 80;
    server_name siceback.maewalliscorp.org;
    root /var/www/sice-backend/public;
    index index.php;

    client_max_body_size 10M;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_read_timeout 120;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

6.2 Configuración del frontend

nano /etc/nginx/sites-available/sice-frontend
server {
    listen 80;
    server_name sice.maewalliscorp.org;
    root /var/www/sice-frontend;
    index index.html;

    gzip on;
    gzip_types text/plain text/css application/json
               application/javascript text/xml
               application/xml image/svg+xml;
    gzip_vary on;

    # Assets con hash (Vite) — cachear 1 año
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SVG e imágenes estáticas — cachear 7 días
    location ~* \.(svg|ico|png|jpg|jpeg|webp|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public";
    }

    # SPA — todas las rutas van a index.html
    location / {
        try_files $uri $uri/ /index.html;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

6.3 Activar sitios y verificar

ln -sf /etc/nginx/sites-available/sice-backend  /etc/nginx/sites-enabled/
ln -sf /etc/nginx/sites-available/sice-frontend /etc/nginx/sites-enabled/

# Eliminar el sitio por defecto si existe
rm -f /etc/nginx/sites-enabled/default

# Verificar sintaxis y recargar
nginx -t && systemctl reload nginx
7
Certificados SSL (Let's Encrypt)
⚠️
Los DNS de ambos dominios deben estar apuntando a la IP del VPS y propagados antes de ejecutar Certbot. Si los DNS no están propagados, el reto HTTP-01 fallará.

7.1 Obtener certificados

certbot --nginx \
  -d siceback.maewalliscorp.org \
  -d sice.maewalliscorp.org \
  --non-interactive \
  --agree-tos \
  --email edgar.degante.a@gmail.com \
  --redirect

Certbot modifica automáticamente los bloques de Nginx para agregar HTTPS y redirección 80→443.

7.2 Verificar renovación automática

certbot renew --dry-run
✅
La renovación automática está configurada por Certbot vía un timer de systemd. Los certificados se renuevan 30 días antes de expirar sin intervención manual.
8
Servicio Gotenberg (PDFs)

8.1 Descargar imagen Docker

docker pull gotenberg/gotenberg:8

8.2 Crear servicio systemd

cat > /etc/systemd/system/sice-gotenberg.service << 'EOF'
[Unit]
Description=Gotenberg PDF Service para SICE
After=docker.service
Requires=docker.service

[Service]
Restart=always
RestartSec=5
ExecStartPre=-/usr/bin/docker stop sice-gotenberg
ExecStartPre=-/usr/bin/docker rm sice-gotenberg
ExecStart=/usr/bin/docker run --name sice-gotenberg \
  -p 127.0.0.1:3000:3000 \
  gotenberg/gotenberg:8 \
  gotenberg \
  --chromium-disable-javascript=false \
  --chromium-allow-list=file:///.*
ExecStop=/usr/bin/docker stop sice-gotenberg

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable sice-gotenberg
systemctl start sice-gotenberg

8.3 Verificar que Gotenberg está activo

sleep 8 && curl http://localhost:3000/health

Respuesta esperada: {"status":"up"} o similar.

# Gotenberg
journalctl -u sice-gotenberg -f
